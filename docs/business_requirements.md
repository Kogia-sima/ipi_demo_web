# 間接プロンプトインジェクションを体験できるサイト ビジネス要件定義書

## 1. プロジェクト概要

### 1.1 背景

近年、生成AIおよびAIエージェントの普及に伴い、**間接プロンプトインジェクション（Indirect Prompt Injection）**と呼ばれる新たなセキュリティ脅威が顕在化している。この攻撃手法は、外部のWebサイト、MCPサーバー、スキルなどに悪意ある指示を仕込み、AIに意図しない動作をさせるものであり、個人情報や機密情報の漏洩につながる可能性がある。

しかし、この脅威は**抽象的かつ技術的な内容**であるため、非エンジニアを含む一般ユーザにはその危険性が伝わりにくいという課題がある。

### 1.2 目的

本プロジェクトでは、間接プロンプトインジェクションという攻撃手法を、**非エンジニアを含む幅広い層が視覚的かつ体験的に理解できる教育用Webサイト**を構築する。

### 1.3 ターゲットユーザ

- 非エンジニアを含む**幅広いインターネットユーザ**
- AIエージェントを業務や日常で利用している、または利用を検討している層
- セキュリティ意識を高めたい一般ユーザ

### 1.4 プロジェクトの特徴

- 実際のAIや外部APIは使用せず、**フロントエンドのみで完結する模擬体験コンテンツ**として実装する
- 学習効果を高めるため、**本物のAIチャットサービス（Gemini）に近いリアルな見た目**を採用する
- 専門用語を極力排除し、**日本語の疑似ログのみで仕組みを説明**する

## 2. ビジネス要件

### 2.1 提供する学習体験

#### 2.1.1 体験フロー

1. ユーザがサイトにアクセスすると、**免責事項のモーダル**が表示される（初回アクセス時のみ）
2. モーダルを閉じると、**チャット画面が表示**され、3つのプロンプトから1つを選択できる
3. プロンプトを送信すると、**チャット画面にAIの応答が表示**されると同時に、**右側に疑似ログがリアルタイムで流れる**
4. 攻撃が発生するタイミングで、**疑似ログの該当箇所が赤色で強調**され、**世界地図上で情報送信のアニメーション**が再生される
5. 一連の動作が完了すると、**解説ポップアップが表示**され、攻撃の仕組みと対策を学べる
6. ユーザは**3つのシナリオを自由な順序で何度でも体験**できる

#### 2.1.2 体験設計上の方針

- 各シナリオの所要時間は**おおむね30〜60秒**を目安とする
- ユーザが任意のタイミングでシナリオを**中断できる**ようにする
- シナリオを**順番に体験する必要はなく**、自由に選択可能とする

### 2.2 3つの攻撃シナリオ

#### 2.2.1 シナリオ1：Webサイト要約（悪意のあるWebサイトによる攻撃）

| 項目 | 内容 |
| --- | --- |
| ユーザのプロンプト | 次のWebサイトを要約して: https://news.ichigo.com/123 |
| 攻撃の仕掛け場所 | 外部Webサイトのコンテンツ内に巧妙に仕込まれた指示 |
| 窃取される情報 | ユーザのGoogle Driveへのアクセスとパスワード |
| 送信元 | 日本 |
| 送信先 | 東欧地域（架空の攻撃者拠点） |

#### 2.2.2 シナリオ2：天気予報取得（悪意のあるMCPサーバーによる攻撃）

| 項目 | 内容 |
| --- | --- |
| ユーザのプロンプト | 明日の天気予報を取得して |
| 攻撃の仕掛け場所 | MCPサーバーが返すデータに仕込まれた指示 |
| 窃取される情報 | ユーザの現在位置と行動履歴 |
| 送信元 | 日本 |
| 送信先 | 東南アジア地域（架空の攻撃者拠点） |

#### 2.2.3 シナリオ3：会議資料作成（悪意のあるスキルによる攻撃）

| 項目 | 内容 |
| --- | --- |
| ユーザのプロンプト | 今日の会議資料を作成して |
| 攻撃の仕掛け場所 | AIが読み込むスキル定義に仕込まれた指示 |
| 窃取される情報 | 会社のフォルダ内の機密ファイル |
| 送信元 | 日本 |
| 送信先 | 南米地域（架空の攻撃者拠点） |

### 2.3 解説コンテンツ

#### 2.3.1 解説の構成

各シナリオ完了後に表示される解説ポップアップには、以下を含める。

- シナリオ名のタイトル
- **攻撃の仕組みを示すイラスト**（SVGによる図解）
- **イラストの解説文**（非エンジニアでも理解できる平易な日本語）
- **具体的な対策方法**（一般ユーザでも実践可能な内容、箇条書き）

#### 2.3.2 シナリオ別の解説内容

**シナリオ1（Webサイト要約）の解説**

- **攻撃の仕組み**：悪意のあるWebサイトが、人間には見えない指示をページ内に隠し、AIがそれを読み込んで実行してしまう仕組みを図解で説明
- **対策方法**：
  - 信頼できないWebサイトをAIに要約させない
  - AIに不必要な権限（Google Driveへのアクセス権など）を与えない
  - AIの出力結果を必ず確認する
  - 重要な認証情報はAIから隔離された場所に保管する

**シナリオ2（天気予報）の解説**

- **攻撃の仕組み**：MCPサーバーが返すデータに悪意ある指示が混入し、AIがそれを信頼して実行してしまう仕組みを図解で説明
- **対策方法**：
  - 信頼できるMCPサーバーのみを使用する
  - MCPサーバーの提供元を確認する
  - 不要なMCPサーバーは無効化する
  - AIの権限を最小限に保つ

**シナリオ3（会議資料作成）の解説**

- **攻撃の仕組み**：AIが利用するスキル（カスタム機能）に悪意ある指示が仕込まれている仕組みを図解で説明
- **対策方法**：
  - 信頼できる提供元のスキルのみインストールする
  - スキルの権限範囲を確認する
  - 定期的に不要なスキルを削除する
  - 機密ファイルへのアクセス権限を慎重に管理する

### 2.4 データに関する要件

#### 2.4.1 取り扱うデータ

- 本サイトは**ユーザの個人情報を一切収集しない**
- 表示する情報（位置情報、パスワード、機密ファイルなど）はすべて**抽象的な表現**で扱い、具体的な内容は明示しない
- シナリオの進行状況など**ユーザの行動データは保存しない**（リロード時にリセット）

#### 2.4.2 アクセス解析データ

- **Vercel Analytics**を用いて、以下の基本的なアクセス解析データを収集する
  - ページビュー数
  - 訪問者数
  - 参照元
  - デバイス種別
  - 地域情報（国レベル）
- **個人を特定する情報は収集しない**

### 2.5 公開・運用に関する要件

| 項目 | 内容 |
| --- | --- |
| ホスティング | Vercel（無料プラン） |
| ドメイン | Vercelが提供するデフォルトドメインを使用（独自ドメインは使用しない） |
| 対応デバイス | PCを主要ターゲットとし、可能な範囲でモバイル（スマートフォン・タブレット）にも対応 |
| 対応言語 | 日本語のみ |

### 2.6 デザイン・トーン

- **Google Geminiに近いUI**を参考にした、本物のAIチャットサービスに見えるデザイン
- **ライトモードのみ対応**（ダークモードは対象外）
- **真面目で誠実なトーン**を維持

### 2.7 法的・倫理的配慮

#### 2.7.1 免責事項の表示

- **初回アクセス時にモーダルで免責事項を表示**
- 以下の内容を明示する
  - 本サイトは**教育目的のシミュレーション**であり、実際のAIや攻撃は発生しないこと
  - **実在する企業・サービス（Google Drive等）との関係を明確に否定**
- **Cookieに同意済みフラグを記録**し、次回以降は表示しない
- フッターに常時表示の**免責事項リンク**を設置
