# 間接プロンプトインジェクションを体験できるサイト システム要件定義書

## 3. システム要件

### 3.1 機能要件

#### 3.1.1 画面構成

**メイン画面（チャット体験画面）**

- **1画面2カラム構成**
  - **左カラム（チャット画面）**：Geminiライクなチャット UI、画面幅の概ね50%
  - **右カラム（AI内部ログ画面）**：疑似ログ表示と世界地図、画面幅の概ね50%
- **ヘッダー部**：サービス名のロゴ
- **フッター部**：免責事項リンク、GitHubリンク (https://github.com/Kogia-sima/ipi_demo_web)

**免責事項モーダル**

- 初回アクセス時に自動表示
- 「**理解しました**」ボタンの押下で閉じる
- 閉じる際にCookieに同意フラグを記録

**解説ポップアップ**

- シナリオ完了後に自動表示（**シナリオ中断時は表示しない**）
- 「**閉じる**」ボタンで閉じる
- モーダル外クリックでも閉じることが可能

#### 3.1.2 チャット画面の機能

**プロンプト選択機能**

- 画面下部に**3つのプロンプトボタン**を常時表示
  - 「次のWebサイトを要約して: https://news.ichigo.com/123」
  - 「明日の天気予報を取得して」
  - 「今日の会議資料を作成して」
- ボタン押下で**該当プロンプトがチャット入力欄に自動入力**され、即座に送信される
- シナリオ実行中はボタンを**非活性化**

**チャットメッセージ表示機能**

- **ユーザの発言**：右寄せ、Geminiライクな吹き出しデザイン
- **AIの応答**：左寄せ、AIアバターアイコン付き
- **タイピングアニメーション**：AIの応答は1文字ずつ表示される演出
- メッセージは**シナリオ完了まで履歴として残る**

#### 3.1.3 AI内部ログ画面の機能

**疑似ログ表示機能**

- 黒背景で白文字のターミナル風デザイン
- **時系列順**にログが下方向に追加される
- 各ログには以下を含む
  - **タイムスタンプ**（実時間ではなくシナリオ進行に基づく相対表示）
  - **アクション名**（例：「Webサイトにアクセス中」「データを処理中」など）
  - **詳細メッセージ**（日本語、専門用語を避ける）
- **通常のログ**：白の文字色
- **攻撃を示すログ**：**赤色で強調表示**し、警告アイコンを併記
- ログ領域は**自動スクロール**で最新行を表示

**世界地図アニメーション機能**

- ログ画面上に**インライン**で表示
- **シンプルなイラスト調**のSVG世界地図
- **送信元（被害者）と送信先（攻撃者）**の位置をピンで表示
- 攻撃発生時に**情報パケットが送信元から送信先へ飛んでいくアニメーション**
- 送信元は**日本に固定**
- 送信先は**シナリオごとに異なる地域**（シナリオ1：東欧、シナリオ2：東南アジア、シナリオ3：南米）

#### 3.1.4 シナリオ実行制御機能

**シナリオ進行制御**

- プロンプト送信から完了まで、**自動的にシーケンスが進行**
- 各シナリオの所要時間は**おおむね30〜40秒**を目安に設計

**シナリオ中断機能**

- シナリオ実行中、画面に「**中断**」ボタンを常時表示
- 押下時の挙動
  - ログ表示エリアに「**処理を中断しました**」と表示
  - 進行中のアニメーションを即座に停止
  - **解説ポップアップは表示しない**
  - 中断後は3つのプロンプトボタンが再度活性化し、別のシナリオを選択可能

**再実行機能**

- 解説ポップアップを閉じた後、または中断後、**他のシナリオを自由に選択可能**
- 同じシナリオも**何度でも実行可能**

#### 3.1.5 Cookie管理機能

| 項目 | 内容 |
| --- | --- |
| 用途 | 免責事項モーダルの表示制御 |
| キー名 | ipi_demo_disclaimer_accepted |
| 値 | true |
| 有効期限 | 1年間 |
| SameSite属性 | Lax |

- 上記以外のCookieは使用しない

### 3.2 非機能要件

#### 3.2.1 性能要件

| 項目 | 内容 |
| --- | --- |
| 初回ページロード | 3秒以内（一般的な回線速度） |
| シナリオ開始からの応答 | プロンプト送信後1秒以内に疑似ログの表示開始 |
| アニメーション | 60FPSを目標、最低でも30FPSを維持 |
| 画像・SVGリソース | 適切な圧縮と遅延読み込みを実施 |

#### 3.2.2 可用性要件

- **稼働率**：Vercelのデフォルト水準（公称99.99%）に準拠
- **メンテナンス**：原則として無停止デプロイ

#### 3.2.3 セキュリティ要件

- **HTTPSによる通信**：Vercelのデフォルト設定を利用
- **CSP（Content Security Policy）**：適切なヘッダー設定により、外部スクリプトの実行を制限
- **依存パッケージの脆弱性管理**：定期的な`npm audit`の実施を推奨
- **ユーザ入力の取り扱い**：本サイトでは自由入力フォームを設けないため、入力検証の対象は最小限

#### 3.2.4 ユーザビリティ要件

**アクセシビリティ**

- **WCAG 2.1のレベルA**を目標とした基本的な配慮
  - 適切な色コントラスト比
  - キーボード操作への対応
  - スクリーンリーダー向けのalt属性、aria属性の付与
- ただし、攻撃時の赤色強調は意図的なデザインとして保持する

**レスポンシブ対応**

| デバイス | 画面幅 | レイアウト |
| --- | --- | --- |
| デスクトップ | 1024px以上 | 2カラムレイアウトを基本とする |
| タブレット | 768px〜1023px | 2カラムを維持しつつ、要素サイズを調整 |
| モバイル | 767px以下 | 1カラムレイアウトに変更し、タブ切り替え等でチャット画面とログ画面を行き来できるUIとする |

**ブラウザ対応**

- **対応ブラウザ**：Chrome、Edge、Safari、Firefoxの**各最新版およびその1つ前のバージョン**
- **Internet Explorerは非対応**

---

## 4. 制約事項・前提条件

### 4.1 制約事項

- 本サイトは**実際のAIや外部APIを使用しない**フロントエンドのみの実装とする

### 4.2 前提条件

- ユーザは**モダンブラウザ**を利用していること
- ユーザはCookieを**有効にしていること**（無効の場合、免責事項モーダルが毎回表示される）

---

## 5. 対象外事項

以下の項目は本プロジェクトの対象外とする。

- 実際のAIモデル（LLM）の組み込み
- ユーザ認証機能
- ユーザの入力した自由なプロンプトに対する応答（あくまで3つの固定プロンプトのみ）
- 多言語対応（日本語以外）
- ダークモード対応
- シナリオ完了状態の永続化（LocalStorage等）
- SNSシェア機能
- 音響効果（警告音など）
